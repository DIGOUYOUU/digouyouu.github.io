前端 性能优化 JavaScript

前端性能会直接影响首屏速度、用户停留和转化率。页面越快，用户越容易留下来。

先优化加载

代码分割、图片懒加载和缓存策略是最常见也最有效的起点。

// 动态导入，按需加载
const component = () => import('./heavy-component.js');

再优化运行时

  • 避免在主线程执行太重的任务。
  • 及时清理监听器和定时器。
  • 把长任务拆成更小的块。

测量与验证

最快的代码，就是不需要执行的代码。

优化不能靠感觉，应该先测量，再改动，再验证。
