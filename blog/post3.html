Git 版本控制 DevOps

Git 是现代开发里最基础的协作工具之一。它不只是保存历史，更是分支协作、回滚和代码审查的基础设施。

工作流

  • 修改代码后先看状态。
  • 把需要提交的文件加入暂存区。
  • 用清晰的提交信息记录变更。
  • 同步远程分支，尽量减少冲突。

提交规范

feat(auth): add JWT authentication

Implement JWT-based authentication system with refresh token support.

冲突处理

冲突说明多人在同时推进同一个项目。关键是及时解决、清楚标记和小步合并。

总结

好的版本控制，是好的团队协作的基础。
